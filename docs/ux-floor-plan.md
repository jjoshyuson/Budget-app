# Budgeting App UX Floor Plan

## Product promise

Log spending in a few seconds and see how the month is going immediately.

## Primary screens

### Home
- This month spend
- Remaining budget
- Mini category progress strip
- Recent transactions
- Primary quick-add entry point

### Quick Add
- Amount-first keypad
- Default expense mode
- Fast category choice
- Optional merchant, note, and date behind one expand action

### Transactions
- Search
- Fast edit
- Delete
- Month-period list

### Budget
- Category budgets
- Progress bars
- Under / near / over states

### Insights
- Monthly spend vs income totals
- Category breakdown
- Simple daily activity bars

### Settings
- Currency
- Locale
- Budget start day
- Category budget editing
- Backup export and restore
- Install help

## UX rules

- The common case is amount + category + save.
- The app should never open on a dense analytics screen.
- The first screen should answer where the month stands.
- Every important action should be reachable with a thumb.
- Optional detail should stay collapsed until needed.
- Overspending feedback should be clear, not shaming.
- Empty states should teach the next useful action.
